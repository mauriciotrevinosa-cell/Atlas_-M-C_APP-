# Workflow Flexible

Esto no es ley. Es una guia editable para mantener ritmo:

1. Detectar una necesidad o bloqueo.
2. Hacer un cambio pequeno y claro.
3. Verificar que el cambio sea visible en UI o API.
4. Conectar el cambio con Atlas (flujo completo, no modulo aislado).
5. Registrar brevemente lo que se aprendio.
6. Repetir con la siguiente mejora.

Principio central:
- Priorizar avances conectados y visibles sobre documentos rigidos.
